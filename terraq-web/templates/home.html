Architecture

TerraQ leverages a pipeline architecture combining classical and modern natural language processing techniques.

The TerraQ application consists of four components:

  1. The Web UI serves as the point of interaction for the user. Through it the user can make textual requests in plain English, visualize results and download images.
  2. The TerraQ pipeline is responsible for understanding the user's request and translating it to a machine-executable SPARQL query.
  3. The Knowledge Graph is a purpose-built data repository that contains information about satellite images, administrative divisions, natural features and more.
  4. The RDF engine is tasked with executing the query over the target Knowledge Graph and returning a result.
